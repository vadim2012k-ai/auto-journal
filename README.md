# Автожурнал — прототип

Мобильный веб-журнал обслуживания автомобиля: интерактивная 2D-схема
(вид снизу, заднеприводный автомобиль, мотор спереди) с кликабельными
узлами — двигатель, КПП, редуктор, тормоза, 4 колеса, охлаждение.
По каждому узлу ведётся история: дата, пробег, марка/характеристики,
стоимость, заметка. Статус узла (норма / скоро / просрочено) считается
автоматически по пробегу.

Это **прототип без сервера** — все данные хранятся локально в браузере
(localStorage). Для резервного копирования используйте кнопки
«Экспортировать / Импортировать JSON» на вкладке «Настройки».

## Почему не React/Vite

Целевой стек — React + TypeScript (Vite). Но в момент сборки у меня не было
доступа к npm-реестру (облачная песочница блокирует внешние загрузки), поэтому
я собрал тот же интерфейс на чистом TypeScript + нативных ES-модулях браузера —
без сборщика и без единой внешней зависимости. Код специально разложен по
файлам в стиле компонентов (`view.ts` — рендер, `app.ts` — состояние и
обработчики событий, `store.ts` — данные), чтобы перенос в React занял часы,
а не дни: каждая `render...()`-функция — по сути будущий компонент, `store.ts`
— будущий контекст/стор.

## Как запустить

Нужен только Node.js (18+) — никаких `npm install` не требуется.

```bash
# 1. Скомпилировать TypeScript → JS (один раз, либо --watch при разработке)
npx -y typescript@5 tsc -p .
# если tsc уже установлен глобально — просто:
tsc -p .

# 2. Поднять локальный сервер и открыть в браузере
npx -y http-server . -p 8080 -c-1
# затем откройте http://localhost:8080 (лучше сразу в мобильном режиме DevTools)
```

Если `npx` тоже недоступен — подойдёт любой статический сервер
(`python3 -m http.server 8080`), либо просто откройте `index.html` напрямую
через `file://` (localStorage и ES-модули по `file://` могут работать
нестабильно в некоторых браузерах — сервер надёжнее).

## Структура

```
src/
  types.ts     — модели данных (Car, MaintenanceRecord, ...)
  config.ts    — справочник категорий обслуживания и зон схемы, интервалы
  store.ts     — CRUD поверх localStorage
  status.ts    — расчёт статуса (норма/скоро/просрочено) по пробегу
  diagram.ts   — генерация SVG-схемы авто с кликабельными зонами
  view.ts      — рендер экранов (Гараж/Журнал/Настройки, панели, формы)
  app.ts       — состояние UI + делегирование событий
  main.ts      — точка входа
index.html
styles.css     — мобильный dark-тема интерфейс
```

## Что дальше (открытые вопросы для следующей итерации)

- Периодичности ТО (км) в `config.ts` — ориентировочные, стоит свериться
  с реальным регламентом ваших авто и, возможно, сделать их редактируемыми.
- Пока поддерживается один автомобиль. Если нужно несколько — структура
  данных (`AppData.cars`) уже это позволяет, не хватает только UI переключения.
- SVG-схема — плейсхолдер. Финальную иллюстрацию (и позже — генерацию по
  VIN через нейросеть) можно вставить, заменив содержимое `diagram.ts`,
  сохранив те же `data-zone`-атрибуты.
- Бэкенд/аккаунты/синхронизация между устройствами — следующий этап после
  обкатки прототипа, как договаривались.
